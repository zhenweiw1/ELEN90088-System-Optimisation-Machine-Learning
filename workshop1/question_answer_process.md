# Question 1 学习整理：多项式凸性判断

---

## 1.1 一阶多项式怎么判断凸性

### 题目对象
二元一阶多项式的一般形式为：

`p(x) = c10*x1 + c01*x2 + c00`

这类函数本质上是**仿射函数（affine function）**，也就是“线性函数 + 常数”。

---

### 核心知识点

#### 1. 什么是 Hessian
对于二元函数 `f(x1, x2)`，Hessian 是二阶偏导组成的矩阵：

`H = [[∂²f/∂x1², ∂²f/∂x1∂x2], [∂²f/∂x2∂x1, ∂²f/∂x2²]]`

它描述函数在各个方向上的二阶弯曲情况。

---

#### 2. 为什么 Hessian 可以判断凸凹
一元函数中：

- `f''(x) >= 0`：函数凸
- `f''(x) <= 0`：函数凹

多元函数中没有单个二阶导数，要看所有方向上的二阶弯曲，因此用 Hessian。

对任意方向向量 `z != 0`，二阶弯曲量由

`z^T H z`

给出。

- 若对所有 `z` 都有 `z^T H z >= 0`，则 Hessian 半正定，函数凸
- 若对所有 `z` 都有 `z^T H z <= 0`，则 Hessian 半负定，函数凹

---

#### 3. 什么叫“所有方向”
在二元情形中，点是 `(x1, x2)`，方向是任意非零向量：

`d = (d1, d2), d != 0`

这不只是坐标轴方向，还包括任意斜方向，例如 `(1,1)`、`(2,-1)` 等。

所以判断凸性时，不能只看沿 `x1` 轴或 `x2` 轴的弯曲，而要看平面中所有可能方向。

---

#### 4. 关于 `z^T H z` 的矩阵乘法
若 `z` 是列向量，则：

- `z` 的维度是 `n x 1`
- `z^T` 的维度是 `1 x n`
- `H` 的维度是 `n x n`

因此

`z^T H z`

的维度是：

`(1 x n)(n x n)(n x 1) -> (1 x 1)`

最终结果是一个标量。

注意：

- 可以先算 `(z^T H)` 再乘 `z`
- 也可以先算 `H z` 再左乘 `z^T`
- 但不能随便交换顺序
- 转置 `T` 很重要；若写成 `zHz`，维度通常不匹配，无法计算

---

### 解题步骤

#### 第一步：写出一般形式
`p(x) = c10*x1 + c01*x2 + c00`

#### 第二步：求一阶导
- `∂p/∂x1 = c10`
- `∂p/∂x2 = c01`

#### 第三步：求二阶导
- `∂²p/∂x1² = 0`
- `∂²p/∂x2² = 0`
- `∂²p/∂x1∂x2 = 0`

所以 Hessian 为：

`H = [[0, 0], [0, 0]]`

#### 第四步：判断凸性
零矩阵既半正定，也半负定，因此：

- 一阶多项式一定是凸函数
- 同时也是凹函数

---

### 本小问结论
**任意一阶多项式都是凸函数，也是凹函数。**

---

### 易错点
- 不要把“矩阵元素非负”误认为“矩阵半正定”
- 凸性看的是 Hessian 的整体性质，而不是某个元素的正负
- 一阶函数没有弯曲，所以 Hessian 恒为零矩阵

---

## 1.2 二次多项式怎么判断凸性

### 题目对象
二元二次多项式的一般形式为：

`p(x) = c20*x1^2 + c11*x1*x2 + c02*x2^2 + c10*x1 + c01*x2 + c00`

其中：

- 二次项决定弯曲
- 一次项和常数项不影响凸凹性

---

### 核心知识点

#### 1. 二次函数为什么好判断
因为二次函数的二阶导数是常数，所以 Hessian 是**常矩阵**，不随 `x` 改变。

因此只要这个 Hessian 半正定，就对所有点都成立，函数就是凸的。

---

#### 2. 正确的判定标准
不是看 Hessian 的每个元素是否都大于 0。

真正要看的是：

`H >= 0`（这里表示 Hessian 半正定）

也就是：对所有 `z != 0`，都有 `z^T H z >= 0`。

矩阵里可以有负元素，但整体仍可能是正定或半正定。

例如：

`[[2, -1], [-1, 2]]`

虽然含有负元素，但它仍是正定矩阵。

---

#### 3. 2x2 对称矩阵半正定条件
对矩阵

`[[a, b], [b, d]]`

半正定的常用判定条件为：

1. `a >= 0`
2. `ad - b^2 >= 0`

---

### 解题步骤

#### 第一步：写出一般形式
`p(x) = c20*x1^2 + c11*x1*x2 + c02*x2^2 + c10*x1 + c01*x2 + c00`

#### 第二步：求一阶导
- `∂p/∂x1 = 2*c20*x1 + c11*x2 + c10`
- `∂p/∂x2 = c11*x1 + 2*c02*x2 + c01`

#### 第三步：求 Hessian
`H = [[2*c20, c11], [c11, 2*c02]]`

注意：该矩阵与 `x` 无关，是常矩阵。

#### 第四步：判断 Hessian 是否半正定
令：

- `a = 2*c20`
- `b = c11`
- `d = 2*c02`

根据 `2x2` 对称矩阵半正定条件：

条件 1：
`2*c20 >= 0`
所以
`c20 >= 0`

条件 2：
`(2*c20)*(2*c02) - c11^2 >= 0`

也就是：
`4*c20*c02 - c11^2 >= 0`

通常还会同时写出：
`c02 >= 0`

---

### 本小问结论
二次多项式凸，当且仅当：

- `c20 >= 0`
- `c02 >= 0`
- `4*c20*c02 - c11^2 >= 0`

等价地说，就是 Hessian

`[[2*c20, c11], [c11, 2*c02]]`

是半正定矩阵。

---

### 直观理解
- `x1^2` 和 `x2^2` 项控制各自方向的向上弯曲
- `x1*x2` 项会引入“扭斜”或“耦合弯曲”
- 若交叉项太强，整体可能变成马鞍面，不再凸

因此不能只看平方项系数非负，还必须限制交叉项大小：

`c11^2 <= 4*c20*c02`

---

### 易错点
- 不能把“矩阵所有元素都大于 0”当成凸性条件
- 不能忽略交叉项 `x1*x2`
- 一次项和常数项不影响 Hessian，因此不影响凸凹性判断

---

## 当前阶段总结

到目前为止可以记住两条最核心结论：

1. **一阶多项式**
   - Hessian 恒为零矩阵
   - 一定既凸又凹

2. **二次多项式**
   - 看 Hessian 是否半正定
   - 在二元情况下可化为系数条件：
     - `c20 >= 0`
     - `c02 >= 0`
     - `4*c20*c02 - c11^2 >= 0`

## 1.3 三次多项式怎么判断凸性

### 题目对象
二元三次多项式的一般形式可写为：

`p(x) = c30*x1^3 + c21*x1^2*x2 + c12*x1*x2^2 + c03*x2^3 + 二次项 + 一次项 + 常数`

判断重点放在三次项，因为高次项决定高阶弯曲行为。

---

### 核心知识点

#### 1. 仍然看 Hessian
判断凸性仍然看 Hessian 是否对所有 `x` 都半正定。

和二次多项式不同：

- 二次多项式的 Hessian 是常矩阵
- 三次多项式的 Hessian 是关于 `x1, x2` 的一次函数矩阵

这意味着 Hessian 的元素会随位置变化。

---

#### 2. 三次多项式的 Hessian 结构
对三次多项式求二阶导后，Hessian 的主要元素是：

- `H11 = 2*c20 + 6*c30*x1 + 2*c21*x2`
- `H12 = c11 + 2*c21*x1 + 2*c12*x2`
- `H22 = 2*c02 + 2*c12*x1 + 6*c03*x2`

其中 `H11` 和 `H22` 都是关于 `x` 的一次函数。

---

#### 3. 为什么三次项必须消失
若函数凸，则 Hessian 必须半正定，因此对角线元素必须满足：

- `H11 >= 0` 对所有 `x`
- `H22 >= 0` 对所有 `x`

但一次函数如果含有变量项，就一定能通过把变量取很大或很小，使它变成负数。

所以要让 `H11` 对所有 `x` 都非负，必须有：

- `c30 = 0`
- `c21 = 0`

同理，要让 `H22` 对所有 `x` 都非负，必须有：

- `c12 = 0`
- `c03 = 0`

---

### 解题步骤

#### 第一步：写出三次多项式一般形式
`p(x) = c30*x1^3 + c21*x1^2*x2 + c12*x1*x2^2 + c03*x2^3 + 二次项 + 一次项 + 常数`

#### 第二步：求 Hessian
得到：

- `H11 = 2*c20 + 6*c30*x1 + 2*c21*x2`
- `H12 = c11 + 2*c21*x1 + 2*c12*x2`
- `H22 = 2*c02 + 2*c12*x1 + 6*c03*x2`

#### 第三步：利用半正定的必要条件
若 Hessian 半正定，则对角线必须非负，所以：

- `H11 >= 0` 对所有 `x`
- `H22 >= 0` 对所有 `x`

#### 第四步：分析一次函数不可能恒非负
因为 `H11` 和 `H22` 都是一次函数，所以若变量系数不为 0，就能取某些 `x` 使它们为负。

因此必须有：

- `c30 = 0`
- `c21 = 0`
- `c12 = 0`
- `c03 = 0`

#### 第五步：退化为二次情形
三次项全部消失后，函数退化成二次多项式，再按第 1.2 小问的条件判断凸性。

---

### 本小问结论
**三次多项式在整个 `R^2` 上凸，当且仅当所有三次项系数都为 0，并且剩余二次部分是凸的。**

也就是：

- `c30 = 0`
- `c21 = 0`
- `c12 = 0`
- `c03 = 0`

并且二次部分满足：

- `c20 >= 0`
- `c02 >= 0`
- `4*c20*c02 - c11^2 >= 0`

---

### 直观理解
三次项属于奇次项，往正方向和负方向延伸时行为相反，不可能一直保持“全局向上弯”。

因此，真正全局凸的三次多项式，不能保留三次项。

---

### 易错点
- 不要只看某一个点的 Hessian
- 必须满足“对所有 `x` 都半正定”
- 不要忽略对角线非负只是半正定的必要条件，但这里已经足够推出三次项必须消失

---

## 当前阶段总结

到目前为止可以记住三条核心结论：

1. **一阶多项式**
   - Hessian 恒为零矩阵
   - 一定既凸又凹

2. **二次多项式**
   - 看 Hessian 是否半正定
   - 二元时等价于：
     - `c20 >= 0`
     - `c02 >= 0`
     - `4*c20*c02 - c11^2 >= 0`

3. **三次多项式**
   - 若要在整个 `R^2` 上凸，三次项必须全部消失
   - 最终退化成二次多项式再判断

## 1.4 如何把三次结论推广到所有奇数次多项式

### 核心结论
若一个二元多项式的次数 `d` 是奇数，且想要在整个 `R^2` 上凸，那么所有次数大于 `2` 的项最终都必须消失。

也就是说：

- 它最后只能退化成二次多项式，或更低次
- 然后再按二次多项式的 Hessian 条件判断凸性

---

### 核心知识点

#### 1. Hessian 会把次数降 2
若 `p(x)` 的次数是 `d`，则 Hessian 中每个元素的次数最多是：

`d - 2`

若 `d` 是奇数，那么 `d - 2` 仍然是奇数。

例如：

- `d = 3` 时，Hessian 元素是一阶
- `d = 5` 时，Hessian 元素是三阶
- `d = 7` 时，Hessian 元素是五阶

---

#### 2. 非零奇数次多项式不可能全局恒非负
奇数次多项式在变量趋向正无穷和负无穷时，符号会朝相反方向变化。

所以一个非零奇数次多项式，不可能在整个实数域上始终满足：

`>= 0`

---

#### 3. 凸函数要求 Hessian 半正定
若 `p(x)` 凸，则对所有 `x` 都必须有：

`H(x)` 半正定

因此 Hessian 的对角线元素必须对所有 `x` 满足：

- `H11(x) >= 0`
- `H22(x) >= 0`

但如果这些对角线元素里保留了奇数次最高项，它们就不可能全局非负。

所以高于 2 次的奇数项必须逐层消失。

---

### 解题步骤

#### 第一步：设多项式次数 `d` 为奇数
设 `d >= 3` 且 `d` 为奇数。

#### 第二步：观察 Hessian 的次数
Hessian 的元素次数为 `d - 2`，仍是奇数。

#### 第三步：利用凸性的必要条件
若函数凸，则 Hessian 半正定，因此其对角线元素必须对所有 `x` 非负。

#### 第四步：排除奇数次最高项
非零奇数次多项式不可能全局非负，因此 Hessian 对角线里的最高奇数次项必须为 0。

#### 第五步：重复此过程
把最高次消掉后，若剩余次数仍大于 2，就继续同样论证。

#### 第六步：得到最终形式
最后只可能剩下：

- 二次项
- 一次项
- 常数项

---

### 本小问结论
**任意奇数次多项式若在整个 `R^2` 上凸，则它必须退化成一个凸二次多项式或仿射函数。**

---

### 易错点
- 不要只说“奇数次不能凸”，严格说法是：**若它在整个空间上凸，则高于 2 次的奇数项必须消失**
- 关键不是原函数本身，而是看 Hessian 对角线能否全局非负

---

## 1.5 四次多项式怎么判断凸性，和二次相比难在哪

### 核心结论
四次多项式与二次多项式不同：

- 二次多项式的 Hessian 是常矩阵
- 四次多项式的 Hessian 是关于 `x1, x2` 的二次多项式矩阵

因此四次情形要检查：

`H(x)` 是否对所有 `x` 都半正定

这比二次情形难得多。

---

### 核心知识点

#### 1. 为什么二次简单
二次多项式的 Hessian 不随 `x` 变化，所以只需要检查一个固定矩阵是否半正定。

---

#### 2. 为什么四次难
四次多项式的 Hessian 会随 `x` 变化，因此不是检查一个固定矩阵，而是检查：

- 对所有 `x`
- 一个变化的矩阵都半正定

这本质上更难。

---

#### 3. 四次项不需要像三次项那样消失
因为四次多项式求两次导以后，得到的是二次式。

而二次式有可能对所有实数都非负，所以四次项可以保留且仍然凸。

例如一元函数：

- `f(x) = x^4`
- `f''(x) = 12*x^2 >= 0`

所以 `x^4` 是凸的。

---

### 一元完整例子

#### 例 1：凸四次函数
`f(x) = x^4`

- `f'(x) = 4*x^3`
- `f''(x) = 12*x^2 >= 0`

所以它在整个 `R` 上凸。

---

#### 例 2：严格凸四次函数
`f(x) = x^4 + x^2 + 1`

- `f''(x) = 12*x^2 + 2 > 0`

所以它在整个 `R` 上严格凸。

---

#### 例 3：凹四次函数
`f(x) = -x^4`

- `f''(x) = -12*x^2 <= 0`

所以它在整个 `R` 上凹。

---

#### 例 4：既不是凸也不是凹
`f(x) = x^4 - x^2`

- `f''(x) = 12*x^2 - 2`

当 `x = 0` 时，`f''(0) = -2 < 0`  
当 `x = 1` 时，`f''(1) = 10 > 0`

因此二阶导有时正、有时负，所以它既不是全局凸，也不是全局凹。

---

### 二元四次完整例子

#### 例 5：凸四次函数
`f(x1, x2) = (x1^2 + x2^2)^2`

展开为：

`f(x1, x2) = x1^4 + 2*x1^2*x2^2 + x2^4`

Hessian 为：

`H = [[12*x1^2 + 4*x2^2, 8*x1*x2], [8*x1*x2, 4*x1^2 + 12*x2^2]]`

其行列式为：

`det(H) = 48*(x1^2 + x2^2)^2 >= 0`

且对角线非负，所以 Hessian 半正定。

因此该函数是凸函数。

---

#### 例 6：严格凸四次函数
`f(x1, x2) = (x1^2 + x2^2)^2 + x1^2 + x2^2`

Hessian 为：

`H = [[12*x1^2 + 4*x2^2 + 2, 8*x1*x2], [8*x1*x2, 4*x1^2 + 12*x2^2 + 2]]`

相当于在上一个半正定 Hessian 上再加 `2I`，因此它对所有点都正定。

所以该函数是严格凸函数。

---

#### 例 7：凹四次函数
`f(x1, x2) = -(x1^2 + x2^2)^2`

其 Hessian 是例 5 的相反数，因此半负定。

所以该函数是凹函数。

---

#### 例 8：既不是凸也不是凹
`f(x1, x2) = x1^4 + x2^4 - 3*x1^2*x2^2`

Hessian 为：

`H = [[12*x1^2 - 6*x2^2, -12*x1*x2], [-12*x1*x2, 12*x2^2 - 6*x1^2]]`

取点 `(x1, x2) = (1,1)`：

`H(1,1) = [[6, -12], [-12, 6]]`

其行列式为：

`36 - 144 = -108 < 0`

所以 Hessian 在该点不定，因此该函数既不是凸，也不是凹。

---

#### 例 9：另一类凸四次函数
`f(x1, x2) = (x1 + x2)^4`

展开为：

`f(x1, x2) = x1^4 + 4*x1^3*x2 + 6*x1^2*x2^2 + 4*x1*x2^3 + x2^4`

Hessian 可写为：

`H = 12*(x1 + x2)^2 * [[1,1],[1,1]]`

因为：

- `12*(x1 + x2)^2 >= 0`
- `[[1,1],[1,1]]` 是半正定矩阵

所以该函数是凸函数。

---

### 本小问结论
**四次多项式判断凸性时，不能像二次那样只检查一个常矩阵；必须检查随 `x` 变化的 Hessian 是否处处半正定，因此难度显著增加。**

---

### 易错点
- 不要把“高于二次就一定不凸”当成结论，四次项可以保留
- 四次困难的根源是 Hessian 依赖 `x`
- “给出例子”时最好用真正含 `x1` 和 `x2` 的多项式，而不只是分离变量形式

---

## Q1 总结补充

### 一阶
- Hessian 为零矩阵
- 一定既凸又凹

### 二次
- Hessian 是常矩阵
- 二元时条件为：
  - `c20 >= 0`
  - `c02 >= 0`
  - `4*c20*c02 - c11^2 >= 0`

### 三次
- 若全局凸，则三次项必须全部消失
- 退化成二次再判断

### 任意奇数次
- 若全局凸，则所有高于 2 次的奇数项必须消失
- 最后退化成二次或一次

### 四次
- Hessian 随 `x` 变化
- 判断处处半正定很难
- 但四次项可以保留且仍然凸

## Q2 相关补充：GP / LP / Monomial / Posynomial 整理

### 1. 决策变量（decision variables）是什么

在优化问题中，**变量**指的是需要由优化器来决定取值的量。

以 Question 2 为例，决策变量是：

- `f`：CPU frequency
- `t_tx`：transmission time

因为题目允许设备调整这两个量，所以它们是优化变量。

与之相对的是**参数**或**常数**，即题目已经给定的量，例如：

- `c1`
- `c2`
- `c3`
- `T`

这些量不需要被优化，只是系统参数。

---

### 2. Monomial 和 Posynomial 的中文含义

- **monomial**：单项式
- **posynomial**：正多项式 / 正系数单项式和 / 正系数广义单项式之和

需要注意的是，这里的含义和中学代数里的“多项式”不完全一样。

---

### 3. Monomial 的定义

在 Geometric Programming (GP) 中，**monomial** 指形如

`f(x) = c x_1^{a_1} x_2^{a_2} ... x_n^{a_n}`

的表达式，其中：

- `c > 0`
- `x_1, ..., x_n > 0`
- 指数 `a_1, ..., a_n` 可以是任意实数

#### 例子
以下都是 monomial：

- `3x^2`
- `2x^{-1}y^3`
- `0.5x^{0.5}y^{-2}`

#### 不是 monomial 的例子
以下不是 monomial：

- `x + y`
- `x^2 + 1`
- `x - y`

原因是 monomial 必须是**单个乘积项**，不能有加减。

---

### 4. Posynomial 的定义

**posynomial** 是若干个 monomial 的和，即

`f(x) = \sum_k c_k x_1^{a_{1k}} x_2^{a_{2k}} ... x_n^{a_{nk}}`

其中每一项都必须满足：

- 系数为正：`c_k > 0`
- 变量为正
- 指数可以是任意实数

#### 例子
以下都是 posynomial：

- `x + y`
- `x^2 + 3x^{-1}`
- `2x^{0.5}y + 7y^{-2}`

#### 不是 posynomial 的例子
以下不是 posynomial：

- `x - y`
- `1 - x`
- `x^2 - 3x + 1`

原因是 posynomial 不允许负系数项。

---

### 5. Posynomial 和普通 polynomial 的区别

普通代数中的 polynomial（多项式）通常要求：

- 指数是非负整数
- 系数可以正也可以负
- 变量可以取任意实数

而 GP 中的 posynomial：

- 允许负指数
- 允许分数指数
- 要求每一项系数为正
- 要求变量为正

因此：

- `x^2 - 3x + 1` 是普通 polynomial，但不是 posynomial
- `x + 2/x` 是 posynomial，但不是普通 polynomial

---

### 6. 为什么 `x+y` 不是 monomial

虽然：

- `x` 是 monomial
- `y` 是 monomial

但是：

- `x+y` 这个**整体表达式**不是 monomial，而是 posynomial

判断时看的是**整个表达式的类型**，不是看能否拆成几个 monomial。

也就是说：

- 一个项：monomial
- 多个 monomial 相加后的整体：posynomial

所以：

- `x`：monomial
- `y`：monomial
- `x+y`：posynomial，不是 monomial

---

### 7. GP 的标准形式

一个标准的 Geometric Program 通常写成：

- **最小化** 一个 posynomial
- **满足** 若干个 `posynomial <= 1`
- 以及若干个 `monomial = 1`
- 并且变量满足 `x > 0`

这几个要求本质上是在限制**表达式结构**，不是在增加新的物理条件。

目的是为了保证问题能够通过变量变换转化成凸优化问题，从而高效求解全局最优解。

---

### 8. 为什么 GP 要求变量 `x > 0`

这是因为 GP 中经常会出现：

- `x^{-1}`
- `x^{0.5}`
- `\log x`

如果变量允许为 0 或负数，这些表达式就可能无定义。

所以 GP 统一要求所有变量都满足：

`x_i > 0`

在 Question 2 中，这对应为：

- `f > 0`
- `t_tx > 0`

---

### 9. 为什么不等式约束写成 `posynomial <= 1`

GP 的标准不等式形式是：

`f_i(x) <= 1`

其中 `f_i(x)` 必须是 posynomial。

很多约束虽然原本不是这个样子，但可以整理成这种标准形式。

例如 Question 2 的时延约束：

`c_3 f^{-1} + t_{tx} <= T`

可以写成：

`(c_3 f^{-1} + t_{tx}) / T <= 1`

左边是 posynomial，右边是 1，因此符合 GP 标准形式。

这里的 `=1` 或 `<=1` 只是标准化写法，并不是说变量本身等于 1。

---

### 10. 为什么等式约束写成 `monomial = 1`

GP 中允许的标准等式约束是：

`g(x) = 1`

其中 `g(x)` 必须是 monomial。

这并不意味着只能有一个变量。  
例如：

- `0.5xy^{-1} = 1`
- `2x^2y = 1`

左边都只有一个乘积项，所以仍然是 monomial。

而下面这些则不是合法的 GP 等式约束：

- `x + y = 1`
- `x^2 + y^2 = 1`

因为左边不是 monomial，而是 posynomial。

需要注意：Question 2 本身并没有必须出现等式约束，这里只是 GP 标准形式的一般说明。

---

## LP 与 GP 的整理

### 11. LP 是什么

**LP（Linear Programming）** 即线性规划。

标准特征是：

- 目标函数是线性的
- 约束也是线性的

典型形式为：

- `min c^T x`
- s.t. `Ax <= b`
- 以及可能有 `Ax = b`
- 通常再加 `x >= 0`

#### LP 约束例子
- `x + 2y <= 5`
- `3x - y = 1`
- `x >= 0, y >= 0`

LP 中不能出现：

- `x^2`
- `1/x`
- `xy`

---

### 12. GP 是什么

**GP（Geometric Programming）** 即几何规划。

它适合处理：

- 幂次
- 乘积
- 倒数

但要求表达式必须符合 monomial / posynomial 的结构。

GP 的标准形式是：

- 最小化一个 posynomial
- 满足 `posynomial <= 1`
- 满足 `monomial = 1`
- 变量为正

---

### 13. 为什么要区分 LP 和 GP

因为它们能处理的数学表达式不同，可用的求解器和理论方法也不同。

#### LP 更适合
- 一次项
- 加减
- 线性约束

#### GP 更适合
- 倒数
- 分数幂
- 变量乘积
- 正系数幂函数结构

区分它们的目的不是背概念，而是判断：

- 问题属于哪一类优化
- 该如何整理目标和约束
- 该用什么求解器
- 是否能保证全局最优

---

### 14. LP 与 GP 的约束区别

#### LP 的约束
通常写成：

- `Ax <= b`
- `Ax = b`
- `x >= 0`

特点是所有项都必须是线性的。

#### GP 的约束
通常写成：

- `posynomial <= 1`
- `monomial = 1`
- `x > 0`

特点是允许非线性，但结构必须特殊。

---

### 15. LP 的常见解题思路

对于 LP，常见步骤是：

1. 识别决策变量
2. 写出线性目标函数
3. 写出线性约束
4. 整理成标准形式
5. 用图解法（低维）、单纯形法、内点法或求解器求解

常见工具：

- `scipy.optimize.linprog`
- `cvxpy`

在二维情况下，LP 的最优解通常出现在可行域边界或顶点。

---

### 16. GP 的常见解题思路

对于 GP，常见步骤是：

1. 识别决策变量，并确认变量为正
2. 判断每一项是 monomial 还是 posynomial
3. 将目标写成 posynomial
4. 将约束整理成 `posynomial <= 1`
5. 必要时利用紧约束消去变量，降维为 1D 问题
6. 用 `cvxpy` 等工具以 `gp=True` 求解

GP 常见于：

- 资源分配
- 功率控制
- 能耗最小化
- 通信系统设计

## Question 2 的标准整理版

### 17. Q2.1 的建模与 GP 说明

#### 决策变量
- `f > 0`：CPU frequency
- `t_{tx} > 0`：transmission time

#### 目标函数
总能耗为：

`E_{total} = E_{comp} + E_{tx} = c_1 f^2 + c_2 t_{tx}^{-1}`

因此优化目标为：

`minimize \quad c_1 f^2 + c_2 t_{tx}^{-1}`

#### 约束
总时延不超过 `T`：

`t_{comp} + t_{tx} <= T`

代入 `t_{comp} = c_3 f^{-1}`，得到：

`c_3 f^{-1} + t_{tx} <= T`

写成 GP 标准形式：

`(c_3 f^{-1} + t_{tx}) / T <= 1`

#### 为什么这是一个 GP
- `c_1 f^2` 是 monomial
- `c_2 t_{tx}^{-1}` 是 monomial
- 两者之和是 posynomial

因此目标函数是 posynomial。

约束左边可写为：

`(c_3/T) f^{-1} + (1/T) t_{tx}`

这是两个 monomial 的和，因此也是 posynomial。

所以该问题满足：

- 最小化一个 posynomial
- 满足一个 `posynomial <= 1`
- 变量为正

故该问题是一个合法的 Geometric Program。

---

### 18. Q2.2 的一维化整理

在能耗最小化问题中，最优解通常会把总时延约束用满，因此假设：

`t_{comp} + t_{tx} = T`

代入 `t_{comp} = c_3/f`，得到：

`t_{tx} = T - c_3/f`

将其代回目标函数：

`E(f) = c_1 f^2 + c_2 \left(T - \frac{c_3}{f}\right)^{-1}`

化简得：

`E(f) = c_1 f^2 + \frac{c_2 f}{fT - c_3}`

#### 有效定义域
由于：

- `f > 0`
- `t_{tx} = T - c_3/f > 0`

所以必须满足：

`T - c_3/f > 0`

即：

`f > c_3/T`

因此一维函数 `E(f)` 的有效定义域为：

`f \in (c_3/T, \infty)`

若题目要求画到 `f=3`，则作图区间可取：

`f \in (c_3/T, 3]`

## 1. 变量、参数、约束到底是什么

在优化问题里：

- **决策变量（variables）**：需要优化器决定的未知量  
- **参数（parameters/constants）**：题目已给定、固定不变的量  
- **约束（constraints）**：限制变量可取值范围的条件（可行域）

以 Q2 为例：

- 变量：`f`（CPU 频率）、`t_tx`（传输时间）
- 参数：`c1, c2, c3, T`
- 约束：`c3/f + t_tx <= T`，且 `f>0, t_tx>0`

---

## 2. LP 和 GP 是什么

## 2.1 LP（Linear Programming，线性规划）

典型形式：

- `min c^T x`
- s.t. `Ax <= b`, `Aeq x = beq`, `x >= 0`

特点：

- 目标线性
- 约束线性
- 不能有 `x^2`、`1/x`、`xy`

---

## 2.2 GP（Geometric Programming，几何规划）

标准形式：

- 最小化一个 **posynomial**
- 满足若干 **posynomial <= 1**
- 满足若干 **monomial = 1**
- 变量满足 `x > 0`

特点：

- 允许幂次、乘积、倒数
- 但表达式必须满足 monomial/posynomial 的结构规则

---

## 3. Monomial 和 Posynomial

## 3.1 Monomial（GP语境）

定义：

`f(x)= c * x1^a1 * x2^a2 * ... * xn^an`

要求：

- `c > 0`
- `x_i > 0`
- 指数 `a_i` 可为任意实数（可负、可分数、可零）
- 必须是**单个项**（不能加减多项）

例子（是 monomial）：

- `3x^2`
- `2x^-1 y^3`
- `0.5 x^0.5 y^-2`

不是 monomial：

- `x+y`
- `x^2+1`
- `x-y`

---

## 3.2 Posynomial

定义：**若干 monomial 的和**

例子（是 posynomial）：

- `x+y`
- `x^2 + 3/x`
- `2x^0.5 y + 7y^-2`

不是 posynomial：

- `x-y`（有减号）
- `1-x`
- `x^2-3x+1`

---

## 3.3 常见误区：`x+y` 为什么不是 monomial

- `x` 是 monomial  
- `y` 是 monomial  
- 但 `x+y` 作为整体是 **posynomial**，不是 monomial

判断看**整体表达式**，不能“拆开后硬当成 monomial”。

---

## 4. GP 标准形式里 `<=1` / `=1` 的含义

- `posynomial <= 1`：GP 标准不等式写法（归一化）
- `monomial = 1`：GP 标准等式写法（也是归一化）

这不表示“变量只能等于 1”，而是表达式可通过移项/除常数整理到右边为 1。  
例如 `x y^-1 = 2` 可改为 `0.5 x y^-1 = 1`。

并且 `monomial = 1` **不等于只能一个变量**，可以有多个变量，只要左侧是一个单项乘积。

---

## 5. 为什么 GP 要这么限制表达式

这些规则的目的不是“增加物理限制”，而是保证：

1. 可做对数变换（log-log transform）
2. 可转成凸优化问题
3. 可高效求全局最优（数值上有保证）

---

## 6. Q2 为什么是 GP 不是 LP

Q2 目标与约束：

- 目标：`c1 f^2 + c2 / t_tx`
- 约束：`c3/f + t_tx <= T`

出现 `f^2`、`1/f`、`1/t_tx`，不是线性，所以不是 LP。  
但它们可写成 monomial/posynomial 结构，因此是 GP。

---

## 7. Q2.2 里为什么写 `t_comp = T - t_tx`

原始约束是：

` t_comp + t_tx <= T `

2.2 使用“最优点约束通常是紧的（tight）”这个常见结论，设：

` t_comp + t_tx = T `

因此移项得到：

` t_comp = T - t_tx `

这不是恒等定义，而是“在最优解处满足等号”。

再由 `t_comp = c3/f` 得到：

` t_tx = T - c3/f `

代回目标：

` E(f) = c1 f^2 + c2 / (T - c3/f) = c1 f^2 + c2 f/(fT-c3) `

定义域来自 `f>0` 和 `t_tx>0`：

` f > c3/T `

## 8.1 LP 常见思路

1. 识别变量
2. 写线性目标
3. 写线性约束

## Question 3 学习整理：Aloha 通信协议中的成功概率优化

---

## 3.1 成功概率 `S` 怎么写

### 题目意思
有 `N` 个发送端。  
每个发送端在每个时隙里，独立地以概率 `p` 发包。

一个时隙要想 **成功**，必须满足：

- 恰好 **1 个** transmitter 发送
- 其余 `N-1` 个都不发送

---

### 推导

先固定某一个发送端 `i`：

- 它发送的概率是 `p`
- 其他 `N-1` 个都不发送的概率是 `(1-p)^{N-1}`

所以“第 `i` 个成功发送”的概率是：

`p(1-p)^{N-1}`

但成功的发送端可以是任意一个，共有 `N` 个，因此总成功概率为：

`S(p) = Np(1-p)^{N-1}`

---

### 本小问结论

**一个时隙是 successful slot 的概率为：**

`S(p) = Np(1-p)^{N-1}`

---

## 3.2 如何最大化成功概率

### 1. 优化变量
唯一的决策变量是：

`p ∈ [0,1]`

---

### 2. 优化目标

要最大化成功概率：

`max_{0 <= p <= 1} S(p) = Np(1-p)^{N-1}`

---

### 3. 一阶导数

对

`S(p) = Np(1-p)^{N-1}`

求导：

`S'(p) = N(1-p)^{N-2}(1-Np)`

这是最关键的结果。

---

### 4. 驻点条件

令一阶导数为 0：

`S'(p) = N(1-p)^{N-2}(1-Np) = 0`

得到两种可能：

1. `(1-p)^{N-2} = 0`
   即 `p = 1`
2. `1 - Np = 0`
   即 `p = 1/N`

但注意：

- `p=1` 是边界点，此时所有人都发，几乎一定碰撞，成功概率为 0
- 真正的内部驻点是：

`p^* = 1/N`

---

### 5. 为什么这是最大值

先看边界：

- `S(0)=0`
- `S(1)=0`

再看导数符号：

- 当 `p < 1/N` 时，`1-Np > 0`，所以 `S'(p) > 0`
- 当 `p > 1/N` 时，`1-Np < 0`，所以 `S'(p) < 0`

所以函数先增后减，因此：

`p^* = 1/N`

是全局最大点。

---

### 6. 二阶导数与凸凹性

继续求二阶导：

`S''(p) = N(N-1)(1-p)^{N-3}(Np-2)`

由此可见：

- 当 `Np < 2` 时，`S''(p) < 0`
- 当 `Np > 2` 时，`S''(p) > 0`

因此 `S(p)` **一般不是在整个区间 `[0,1]` 上全局凹函数，也不是全局凸函数**。  
但在最优点附近 `p = 1/N`，有 `Np = 1 < 2`，所以此处二阶导为负，函数局部是凹的，对应局部最大值。

---

### 本小问结论

优化问题为：

`max_{0 <= p <= 1} Np(1-p)^{N-1}`

其最优解为：

`p^* = 1/N`

对应最大成功概率为：

`S^* = N(1/N)(1-1/N)^{N-1} = (1-1/N)^{N-1}`

---

## 3.3 当 `N -> infinity` 时最大成功概率是多少

上一问已经得到：

`S^* = (1-1/N)^{N-1}`

现在令 `N -> infinity`。

利用经典极限：

`lim_{N->infinity} (1-1/N)^N = e^{-1}`

因此：

`lim_{N->infinity} (1-1/N)^{N-1} = e^{-1}`

即：

`S^* -> 1/e ≈ 0.3679`

---

### 本小问结论

当发送端数量趋于无穷时，最大成功概率趋于：

`1/e ≈ 0.368`

---

## Q3 总结

### 1. 成功概率
`S(p) = Np(1-p)^{N-1}`

### 2. 优化问题
`max_{0 <= p <= 1} Np(1-p)^{N-1}`

### 3. 最优发送概率
`p^* = 1/N`

### 4. 最大成功概率
`S^* = (1-1/N)^{N-1}`

### 5. 大规模极限
`lim_{N->infinity} S^* = 1/e`

---

### 易错点

- 不要把“某个节点成功”的概率 `p(1-p)^{N-1}` 直接当成总成功概率，别忘了乘 `N`
- `p=1` 不是最优，虽然也是导数为 0 的边界情况之一
- `S(p)` 一般不是全局凹函数，所以不能直接说“这是凸优化问题”
- 判断最优点最稳的方法是：看 `S'(p)` 的符号变化，加上边界值比较

---
## Answer:

### 3.1

To make a successful transmission, exactly one transmitter must send a frame while the other `N-1` transmitters remain silent. Therefore, the success probability is:

`S(p) = Np(1-p)^(N-1)`

### 3.2

- Decision variable: `p`

- Objective: maximize `S(p) = Np(1-p)^(N-1)`

- Constraint: `0 <= p <= 1`

The first derivative of the objective function is:

`S'(p) = N(1-p)^(N-2)(1-Np)`

Setting the first derivative equal to zero gives:

`N(1-p)^(N-2)(1-Np) = 0`

So the stationary point is:

`p* = 1/N`

To determine whether this is the maximum, observe that:

- when `p < 1/N`, `S'(p) > 0`
- when `p > 1/N`, `S'(p) < 0`

Hence `S(p)` first increases and then decreases, so the optimal solution is:

`p* = 1/N`

The maximal success probability is therefore:

`S* = S(1/N) = N(1/N)(1-1/N)^(N-1) = (1-1/N)^(N-1)`

The second derivative is:

`S''(p) = N(N-1)(1-p)^(N-3)(Np-2)`

Hence, the objective function is generally neither globally convex nor globally concave on `[0,1]`.

### 3.3

At the optimal point `p = 1/N`, the maximal success probability is:

`S* = (1-1/N)^(N-1)`

When `N` tends to infinity,

`lim (N -> infinity) S* = lim (N -> infinity) (1-1/N)^(N-1)`

Using the standard limit

`lim (N -> infinity) (1-1/N)^N = 1/e`

and the identity

`(1-1/N)^(N-1) = (1-1/N)^N / (1-1/N)`

we obtain

`lim (N -> infinity) (1-1/N)^(N-1) = 1/e ≈ 0.3679`

## Question 4 学习整理：无线网络中的功率控制与几何规划

---

## 4.1 如何把问题改写成标准 GP 形式

### 题目对象
有 `N` 个发射器，每个发射器的功率为 `P_i`。

目标是：

- 总功率尽量小
- 每个链路的 SINR 至少达到 `S^min`

原始优化问题为：

`min sum_{i=1}^N P_i`

subject to

- `P^min <= P_i <= P^max`
- `G_ii P_i / (sigma_i + sum_{k != i} G_ik P_k) >= S^min`

---

### 核心知识点

#### 1. 目标函数是 posynomial
目标函数 `sum_i P_i` 是若干个 monomial `P_i` 的和，因此它是 posynomial。

---

#### 2. 上下界约束可以改写成 monomial <= 1
下界：

`P_i >= P^min`

可改写为：

`P^min * P_i^(-1) <= 1`

上界：

`P_i <= P^max`

可改写为：

`P_i * (P^max)^(-1) <= 1`

这两个都是 monomial `<= 1` 约束。

---

#### 3. SINR 约束的关键变形
原始 SINR 约束：

`G_ii P_i / (sigma_i + sum_{k != i} G_ik P_k) >= S^min`

先把分母乘过去：

`G_ii P_i >= S^min (sigma_i + sum_{k != i} G_ik P_k)`

再两边同时除以 `G_ii P_i`：

`1 >= (S^min sigma_i)/(G_ii P_i) + sum_{k != i} (S^min G_ik / G_ii) P_k P_i^(-1)`

整理成标准 GP 形式：

`(S^min sigma_i / G_ii) P_i^(-1) + sum_{k != i} (S^min G_ik / G_ii) P_k P_i^(-1) <= 1`

左边每一项都是 monomial，它们的和就是 posynomial，因此这是一个合法的 `posynomial <= 1` 约束。

---

### 本小问结论
整个问题满足：

- 最小化一个 posynomial
- 约束由 monomial `<= 1` 和 posynomial `<= 1` 组成

因此该问题是一个合法的 **Geometric Program**。

---

### 易错点
- 不要忘记 SINR 约束要先移项，再除以 `G_ii P_i`
- 不要把 `P_k P_i^(-1)` 误以为不是 monomial，它仍然是 monomial
- 关键不是“长得复杂”，而是看能不能整理成 `posynomial <= 1`

---

## 4.2 当 `S^min = 0` 时会发生什么

### 核心知识点

#### 1. SINR 约束会自动满足
当 `S^min = 0` 时，SINR 约束变成：

`G_ii P_i / (sigma_i + sum_{k != i} G_ik P_k) >= 0`

由于：

- `G_ii > 0`
- `P_i > 0`
- `sigma_i + sum_{k != i} G_ik P_k > 0`

所以这个分式本来就非负，因此约束自动成立。

也就是说，此时 SINR 约束 **inactive**，不会影响最优解。

---

#### 2. 优化问题退化成最小化总功率
此时只剩下：

`min sum_{i=1}^N P_i`

subject to

`P^min <= P_i <= P^max`

因为目标是让总和最小，所以每个 `P_i` 都应取最小可行值，即：

`P_i^* = P^min`

在题目给定参数下：

- `N = 10`
- `P^min = 0.1`

因此：

- `P_i^* = 0.1` for all `i`
- 总功率为 `10 * 0.1 = 1.0`

---

#### 3. 为什么还要计算 SINR
虽然 `S^min = 0` 时 SINR 不再限制优化结果，但题目仍要求把最优功率代回公式，观察这组最低功率配置下系统自然产生的 SINR。

因此：

- 左图显示的是所有发射器都取 `0.1`
- 右图显示的是这组最低功率下各链路实际得到的 SINR

---

### 本小问结论
当 `S^min = 0` 时，最优策略就是让每个 transmitter 都取最小功率：

`P_i^* = 0.1`

总功率为：

`sum_i P_i^* = 1.0`

---

### 易错点
- 不要误以为 `S^min = 0` 时 SINR 必须等于 0；这里只是“至少为 0”
- 此时求出来的 SINR 只是结果观察，不是优化要求
- 这一步的重点是理解“约束失效后，最优解退化到边界最小值”

---

## 4.3 当 `S^min` 增大时为什么功率会升高，甚至 infeasible

### 核心知识点

#### 1. `S^min` 表示最低通信质量要求
SINR 约束：

`S_i = signal / (interference + noise) >= S^min`

所以：

- `S^min` 小：要求宽松
- `S^min` 大：要求严格

---

#### 2. 为什么 `S^min` 越大，总功率通常越大
想要让 SINR 变大，通常需要：

- 增大自己的发射功率
- 或减少别人造成的干扰

但无线网络中每个用户都在共享信道，因此大家提高功率的同时也会给别人带来更多干扰。

所以随着 `S^min` 增大，优化器通常必须提高某些或全部用户的功率，导致总功率上升。

---

#### 3. 为什么会 infeasible
题目还限制了：

`P^min <= P_i <= P^max`

也就是说功率不是无限大的。

如果 `S^min` 高到一定程度，就可能出现：

- 即使所有用户都尽量提高功率
- 也无法让所有链路同时满足目标 SINR

这时问题就变成 **infeasible**。

---

### 从数值结果得到的现象
- 当 `S^min` 较小时，解可能仍接近最小功率分配
- 当 `S^min` 继续增大时，总功率明显上升
- 当 `S^min` 太大时，求解器会返回 `infeasible`

这反映的是无线系统里典型的 **功率—性能 trade-off**：

更高的 SINR 要求意味着更高的总功率消耗，而当要求过高时，系统能力会达到上限。

---

### 本小问结论
可以概括为：

`larger S^min -> larger required total power -> eventually infeasibility`

---

### 易错点
- 不要把 infeasible 理解成“代码错了”，它表示“题目要求本身做不到”
- 功率增大并不一定总能无限提升系统性能，因为干扰也在同步增大
- 这一问的关键不是精确数值，而是理解趋势和物理意义

---

## 4.4 修改后的 SINR 为什么仍然是 GP，但不再是 LP

### 题目变化
新的 SINR 形式是：

`S_i = G_ii P_i / (0.5 sum_{j != i}(G_ij P_j sum_{k != i, k != j} G_ik P_k) + sigma_i)`

和 4.1 相比，最大的区别是分母里出现了变量乘积，例如：

`P_j * P_k`

---

### 核心知识点

#### 1. 为什么它仍然是 GP
将约束移项并除以 `G_ii P_i` 后，可得到类似：

`(S^min sigma_i / G_ii) P_i^(-1) + sum sum (0.5 S^min G_ij G_ik / G_ii) P_j P_k P_i^(-1) <= 1`

其中：

- 第一项是 monomial
- 每个 `P_j P_k P_i^(-1)` 也是 monomial

因为 monomial 允许：

- 多个变量相乘
- 任意实数指数
- 包括负指数

所以左边仍然是若干 monomial 的和，也就是 posynomial。

因此约束仍然是合法的：

`posynomial <= 1`

所以问题仍然是 GP。

---

#### 2. 为什么它不再是 LP
LP 要求表达式必须是线性的，也就是变量只能一次出现，不能互相相乘。

但这里已经出现：

- `P_j P_k`
- `P_j P_k P_i^(-1)`

这些都不是线性项，因此这个问题不再是 LP。

---

### 本小问结论
4.4 的新问题：

- **仍然是 GP**，因为约束仍可整理成 `posynomial <= 1`
- **不再是 LP**，因为干扰项中出现了变量乘积

---

### 易错点
- 不要把“非线性”直接等同于“不能做 GP”；很多非线性项仍然是合法 monomial
- 判断是不是 LP 时，最关键就是看有没有变量乘变量
- 判断是不是 GP 时，关键是看能不能整理成 monomial / posynomial 结构

---

## Q4 总结

### 4.1
- 目标函数是 posynomial
- 上下界约束可改成 monomial `<= 1`
- SINR 约束可整理成 posynomial `<= 1`
- 因此原问题是 GP

### 4.2
- 当 `S^min = 0` 时，SINR 约束自动成立
- 最优解是所有发射器都取最小功率 `P^min`

### 4.3
- `S^min` 越大，要求越严格，总功率通常越大
- 若要求过高，则问题会变成 infeasible

### 4.4
- 修改后的约束中出现 `P_j P_k` 乘积项
- 因此不再是 LP
- 但这些项仍是 monomial，所以整体仍可写成 GP

## Question 5 学习整理：Logistic Regression 的梯度下降

---

## 5.1 如何推导经验风险的梯度

### 题目对象
题目给的是二分类 logistic regression。

每个样本为：

- 特征 `x_i in R^d`
- 标签 `y_i in {0,1}`

模型写成：

`log ( p(y=1|x) / (1-p(y=1|x)) ) = x^T w`

其中 `w` 是待学习的参数向量。

---

### 核心知识点

#### 1. logistic / sigmoid 函数
由 log-odds 关系可得：

`mu_i(w) = 1 / (1 + exp(-x_i^T w))`

这就是 sigmoid 函数：

`sigma(z) = 1 / (1 + exp(-z))`

所以 `mu_i(w) = sigma(x_i^T w)`。

---

#### 2. 经验风险是什么
题目中的经验风险（负对数似然 / 交叉熵）是：

`R_hat(w) = -(1/n) sum_i [ y_i log mu_i(w) + (1-y_i) log(1-mu_i(w)) ]`

这个目标越小，说明模型预测概率和真实标签越一致。

---

#### 3. 为什么梯度会变成 `mu_i - y_i`
对单个样本的 loss 求导时，用到两层链式法则：

1. 先对 `log mu_i` 和 `log(1-mu_i)` 求导
2. 再对 `mu_i(w)` 求导

其中：

`d mu_i / d w = mu_i (1-mu_i) x_i`

把这些项整理后，最后会非常漂亮地化成：

`(mu_i - y_i) x_i`

所以总梯度是：

`grad R_hat(w) = (1/n) sum_i (mu_i - y_i) x_i`

矩阵形式写成：

`grad R_hat(w) = (1/n) X^T (mu - y)`

---

### 解题步骤

#### 第一步：写出预测概率
`mu_i(w) = 1 / (1 + exp(-x_i^T w))`

#### 第二步：写出经验风险
`R_hat(w) = -(1/n) sum_i [ y_i log mu_i + (1-y_i) log(1-mu_i) ]`

#### 第三步：对 `w` 求梯度
得到：

`grad R_hat(w) = (1/n) sum_i (mu_i - y_i) x_i`

#### 第四步：写出梯度下降更新式
`w_{t+1} = w_t - eta * grad R_hat(w_t)`

也就是：

`w_{t+1} = w_t - (eta/n) sum_i (mu_i(w_t) - y_i) x_i`

---

### 本小问结论

- 预测概率：`mu_i(w) = 1 / (1 + exp(-x_i^T w))`
- 梯度：`grad R_hat(w) = (1/n) sum_i (mu_i - y_i) x_i`
- 梯度下降：`w_{t+1} = w_t - eta * grad R_hat(w_t)`

---

### 易错点
- 不要忘记经验风险前面有负号
- 不要把标签 `y_i` 写成 `{-1,1}`，这题里是 `{0,1}`
- 梯度最后是 `mu_i - y_i`，不是 `y_i - mu_i`

---

## 5.2 如何从零实现 Logistic Regression

### 题目要求在做什么
题目要求你自己写一个 batch gradient descent 版本的 logistic regression，而不是直接调用 sklearn。

核心包括四部分：

1. `risk(X, y, w)`：计算经验风险
2. `grad_risk(X, y, w)`：计算梯度
3. `fit_logistic(...)`：执行梯度下降
4. 画出数据点和决策边界

---

### 核心知识点

#### 1. 为什么要单独写 `risk`
`risk` 用来计算当前参数 `w` 下模型的损失值，方便：

- 检查训练是否在下降
- 判断模型是否收敛得合理

实现时通常会对 `mu` 做 clipping，例如限制在 `[1e-15, 1-1e-15]`，避免出现 `log(0)`。

---

#### 2. 为什么要单独写 `grad_risk`
梯度下降的核心就是反复用梯度更新参数，所以把梯度单独写成函数最清晰。

在这题中：

`grad_risk(X, y, w) = (1/n) X^T (mu - y)`

---

#### 3. 梯度下降在做什么
每次迭代：

- 先用当前 `w` 算出梯度
- 然后朝负梯度方向走一步

更新式为：

`w_next = w - lr * grad_risk(X, y, w)`

其中 `lr` 是 learning rate。

---

#### 4. 为什么停止条件用无穷范数
题目要求：

`|| grad R_hat(w_t) ||_inf <= tol`

这表示：

- 梯度每个分量都已经很小
- 说明已经接近驻点

所以可以停止迭代。

---

### 解题步骤

#### 第一步：生成数据并加 bias 项
代码里用 `make_classification` 生成二维二分类数据。

然后构造：

`X_b = [1, x1, x2]`

这里前面的常数列 1 用来吸收 bias / intercept。

---

#### 第二步：实现 sigmoid
为了数值稳定，通常不直接无脑写 `1/(1+exp(-z))`，而是用稳定版写法，避免 `z` 很大或很小时溢出。

---

#### 第三步：实现风险函数
`risk(X,y,w)` 计算：

`-(1/n) sum_i [ y_i log mu_i + (1-y_i) log(1-mu_i) ]`

---

#### 第四步：实现梯度函数
`grad_risk(X,y,w)` 返回：

`(1/n) X^T (mu-y)`

---

#### 第五步：实现更新函数
`update_weight_gd` 执行一步：

`w_next = w - lr * grad_risk(X,y,w)`

---

#### 第六步：实现 `fit_logistic`
主要流程：

1. 初始化 `w_0 = 0`
2. 循环计算梯度
3. 若 `||grad||_inf <= tol` 则停止
4. 否则按梯度下降更新
5. 保存 `w_history`

---

#### 第七步：画决策边界
对于二维特征，决策边界由：

`w0 + w1*x1 + w2*x2 = 0`

可写成：

`x2 = -(w0 + w1*x1) / w2`

再和数据点一起画出来，就能看到分类结果。

---

### 本小问结论
从零实现 logistic regression 的核心就是：

- 正确写出 `risk`
- 正确写出 `grad_risk`
- 用梯度下降反复更新参数
- 用梯度足够小作为停止条件

---

### 易错点
- 忘记在 `X` 前面加一列常数 1，导致没有 bias
- `log(0)` 数值错误，需要 clip `mu`
- 决策边界公式写错号，正确是 `x2 = -(w0 + w1*x1)/w2`
- 学习率太大可能发散，太小则收敛很慢

## 5.3 为什么要和 scikit-learn 对比

### 核心知识点

题目要求把自己实现的 logistic regression 和 sklearn 的结果进行比较，目的是验证：

- 数学推导是否正确
- 代码实现是否正确
- 梯度下降是否真的收敛到了同样的解

---

#### 1. 为什么要关掉正则化
sklearn 的 `LogisticRegression` 默认带有正则化项。

但你手写的版本没有加正则化，所以为了公平比较，必须把 sklearn 的正则化关掉。

这就是为什么代码里设置：

`penalty=None`

---

#### 2. 为什么 `fit_intercept=False`
因为我们已经在 `X_b` 里手动加入了一列常数 1，bias 已经作为参数向量的一部分来学习了。

所以 sklearn 不需要再额外拟合一个 intercept。

---

#### 3. 怎么判断实现正确
比较：

- 手写模型得到的权重 `w_manual`
- sklearn 得到的权重 `w_sklearn`

如果两者非常接近（例如差距只有 `1e-8`、`1e-6` 量级），就说明实现是正确的。

---

### 本小问结论
若手写实现与 sklearn 在关闭正则化后得到几乎相同的参数向量，则可以认为：

- 梯度推导正确
- 代码实现正确
- 训练过程正确

---

### 易错点
- 忘记关闭正则化，导致结果不一致
- 忘记设置 `fit_intercept=False`，导致 bias 被重复计算
- 不能只看分类结果相似，最好直接比较参数向量
